## Poll Posts (Browser Automation)

When creating poll posts via browser automation on Skool, use this exact sequence:

### Technical Flow
1. Navigate to community URL (e.g. `https://www.skool.com/buildroom`)
2. Click "Write something..." to open composer
3. Click the poll toolbar button (5th icon in toolbar row) to add poll fields
4. Set title via `input[placeholder="Title"]` using native React value setter
5. Set poll options via `input[placeholder="Option N"]` using native React value setter
6. Set body text via the `[contenteditable] p` or `[data-placeholder]` element
7. **CRITICAL â€” Category selection:** The dropdown button doesn't work with normal clicks. You MUST:
   - Find the "Select a category" button
   - Access its React fiber via `__reactFiber$` key
   - Walk up the fiber tree to find `memoizedProps.options`
   - Call `options[INDEX].onClick()` where INDEX maps to:
     - 0 = Announcements, 1 = Daily Actions, 2 = $1M Roadmap, 3 = YouTube Videos
     - 4 = Wins, **5 = Hangout**, 6 = Check-Ins, 7 = Help!
8. Click Post button
9. Verify by checking if URL changed (redirects to new post page)

### Poll Content Rules
- 3-4 options (never more than 4)
- Options should be short (under 50 chars each)
- Options should be mutually exclusive, easy to pick
- Body text: 1-2 sentences, always end with "drop your answer in the comments" or similar
- Category: Hangout for Build Room polls
