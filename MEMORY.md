## Modular Architecture First (Feb 2026)

**Standing directive:** Always work modular. No more monolithic scripts.

**Process (every task):**
1. Identify repeatable subtasks
2. Create single-purpose modules (<150 lines)
3. Build config-driven orchestrators
4. Each module does ONE thing well

**Structure:**
- `core_function.py` - Reusable utility (e.g., `nano_banana.py`)
- `config.json` - Data, not code
- `orchestrator.py` - Calls utilities in sequence
- `README.md` - Documentation

**Benefits:**
- Sub-agents load only what they need
- Easy debugging (swap individual pieces)
- No API call duplication
- Faster iteration (change config, not code)

**Locked in AGENTS.md** as permanent operating procedure

## Format Validation System (Feb 2026)

**Created:** Feb 14, 2026 in response to Carousel #002 format mismatch
**Problem:** Delivered 9:16 format (1080x1920) when 4:5 format (1080x1350) was required
**Solution:** Built automated validation system

**Utilities:**
1. **format_validator.py** (143 lines) - Validates against gold standards
   - Dimensions (exact match check)
   - Aspect ratio detection (4:5, 9:16, 16:9)
   - Text length validation (catches verbose filler)
   - Copy specificity check (flags vague language)
   
2. **resize_to_format.py** (60 lines) - Converts to proper format
   - 4:5 (1080x1350 Instagram/Twitter)
   - 9:16 (1080x1920 TikTok)
   - 16:9 (1920x1080 YouTube/LinkedIn)
   - Center crop, LANCZOS resampling

**Workflow (MANDATORY):**
1. Generate slides
2. **Validate format** (format_validator.py) ‚Üê BEFORE upload
3. Fix if needed (resize_to_format.py)
4. Upload to hosting
5. Update Notion

**Usage:**
```bash
# Validate entire carousel
python3 format_validator.py carousel-002-final/

# Convert to Instagram format
python3 resize_to_format.py input.png output.png 4:5
```

**Location:** `/data/.openclaw/workspace/carousel-system/`
